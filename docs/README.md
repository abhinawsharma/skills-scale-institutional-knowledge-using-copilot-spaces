# OctoAcme Project Management Docs

This README provides a brief overview of the project management processes used by OctoAcme and links to all process documentation in this folder.

## Overview

OctoAcme follows a structured, iterative project management approach that spans the full project lifecycle—from initiation through retrospective and continuous improvement. The approach emphasizes customer-first delivery, clear ownership, data-informed decisions, and psychological safety.

**Workflows** are organized around sprint-based execution using GitHub Projects for tracking. Teams work in small, testable increments with defined acceptance criteria, pull request conventions, and automated CI checks. Escalation paths are clearly defined at team, PM, product lead, and sponsor levels, ensuring blockers are resolved quickly.

**Roles and personas** are clearly defined to support accountability and collaboration. Core roles include Project Managers (who coordinate delivery, schedules, risk, and communications), Product Managers (who own the product vision and backlog prioritization), Developers (who implement features and maintain quality), and QA/Testing (who validate acceptance criteria). Each persona has documented responsibilities, goals, and typical communication patterns.

**Communication strategies** include daily standups, weekly delivery syncs, monthly stakeholder updates, and structured templates for status reports, incidents, and escalations. A single source of truth (project README or release doc) is maintained for status to ensure transparency across all stakeholder groups.

**Quality assurance** practices include unit and integration tests, end-to-end smoke tests for critical flows, security scanning in CI, and manual QA for feature acceptance. Every release goes through a pre-release checklist before deployment, with rollback and incident playbooks on hand.
